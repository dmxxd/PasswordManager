#  Описание прецедентов

| Прецедент | Аутентификация |
|-----------|----------------|
| **ID** | 1 |
| **Название** | Аутентификация |
| **Цель** | Получить доступ к зашифрованной базе паролей. |
| **Актор** | Пользователь |
| **Основной поток** | 1. Запуск приложения.<br>2. Ввод мастер-пароля.<br>3. Дешифровка и загрузка базы данных.<br>4. Открытие главного окна приложения. |
| **Исключения** | • **Неверный пароль:** Отображение сообщения об ошибке, принудительная задержка 5 секунд перед следующей попыткой.<br>• **Отсутствие файла базы:** Предложение пользователю создать новую базу паролей. |

---

| Прецедент | Добавление записи |
|-----------|-------------------|
| **ID** | 2 |
| **Название** | Добавление записи |
| **Цель** | Сохранить новую учетную запись (логин/пароль) в зашифрованной базе. |
| **Актор** | Пользователь |
| **Основной поток** | 1. Нажатие кнопки «Добавить».<br>2. Выбор существующей категории или создание новой.<br>3. Ввод названия, логина, пароля и опциональных заметок.<br>4. Шифрование и сохранение записи в базу. |
| **Исключения** | • **Пустые обязательные поля:** Предупреждение о необходимости заполнения.<br>• **Дубликат названия:** Уведомление о конфликте с предложением переименовать новую запись. |

---

| Прецедент | Генерация пароля |
|-----------|------------------|
| **ID** | 3 |
| **Название** | Генерация пароля |
| **Цель** | Создать криптографически стойкий пароль. |
| **Актор** | Пользователь |
| **Основной поток** | 1. Открытие диалога/окна генератора паролей.<br>2. Настройка параметров (длина, использование заглавных/строчных букв, цифр, специальных символов).<br>3. Генерация пароля и его копирование в буфер обмена. |
| **Исключения** | • **Недопустимые параметры:** Автоматический сброс параметров на безопасные значения по умолчанию. |

---

| Прецедент | Поиск записи |
|-----------|--------------|
| **ID** | 4 |
| **Название** | Поиск записи |
| **Цель** | Быстро найти сохраненную учетную запись по названию, логину или другим данным. |
| **Актор** | Пользователь |
| **Основной поток** | 1. Ввод текста запроса в поисковую строку.<br>2. Динамический фильтр и отображение подходящих записей.<br>3. Выбор нужной записи из результатов для просмотра, редактирования или копирования данных. |
| **Исключения** | • **Нет результатов:** Отображение информационного сообщения «Ничего не найдено». |

---

| Прецедент | Управление категориями |
|-----------|------------------------|
| **ID** | 5 |
| **Название** | Управление категориями |
| **Цель** | Организовать записи по категориям для удобства навигации. |
| **Актор** | Пользователь |
| **Основной поток** | 1. Открытие раздела управления категориями.<br>2. Выполнение операций: создание, переименование, удаление категорий.<br>3. Перемещение записей между категориями через Drag&Drop или меню. |
| **Исключения** | • **Удаление категории, содержащей записи:** Все записи автоматически перемещаются в категорию по умолчанию («Без категории»). |
