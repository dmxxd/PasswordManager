# Макетирование интерфейса пользователя

В рамках проектирования пользовательского интерфейса были разработаны макеты основных экранов системы. Макеты отражают структуру окон, состав элементов управления и предполагаемую логику взаимодействия пользователя с приложением.

---

## Макет окна аутентификации (LoginForm)

<img width="600" alt="Макет окна аутентификации" src="Макет окна аутентификации.png" />

**Назначение:**  
Окно предназначено для входа пользователя в систему.

**Элементы интерфейса:**
- `Label` — подпись к полю ввода.
- `TextBox` — поле ввода учетных данных.
- `CheckBox` — дополнительная опция (например, отображение пароля).
- `Button` — кнопки входа и отмены.

**Логика взаимодействия пользователя:**
1. Пользователь вводит учетные данные.
2. При необходимости активирует чекбокс.
3. Нажимает кнопку входа.
4. При успешной проверке открывается главное окно приложения.
5. В случае ошибки отображается сообщение об ошибке.

---

## Макет главного окна приложения (MainForm)

<img width="900" alt="Макет главного окна" src="Макет главного окна.png" />

**Назначение:**  
Главное окно предназначено для работы с данными и навигации по функциям системы.

**Элементы интерфейса:**
- Панель навигации с кнопками быстрого доступа.
- Поле поиска и кнопка выполнения действия.
- `DataGridView` для отображения списка записей.
- Кнопки управления записями.

**Логика взаимодействия пользователя:**
1. Пользователь попадает в главное окно после авторизации.
2. Выбирает раздел с помощью панели навигации.
3. Просматривает данные в таблице.
4. Использует поиск или фильтрацию.
5. Запускает операции добавления, редактирования или удаления записей.

## Макет окна добавления / изменения записи (EntryForm)

<img width="600" alt="Макет окна добавления и изменения записи" src="Макет окна добавления-изменения записи.png" />

**Назначение:**  
Окно используется для создания новых записей и редактирования существующих.

**Элементы интерфейса:**
- `Label` и `TextBox` для ввода данных.
- `ComboBox` для выбора значения из списка.
- Многострочное текстовое поле для описаний.
- Кнопки сохранения и отмены.

**Логика взаимодействия пользователя:**
1. Окно открывается из главного окна.
2. Пользователь вводит или изменяет данные.
3. Нажимает кнопку сохранения.
4. Выполняется проверка введённых данных.
5. При успешном сохранении окно закрывается, данные обновляются.

## Макет окна генератора пароля (PasswordGeneratorForm)

<img width="600" alt="Макет окна генератора пароля" src="Макет окна генератора пароля.png" />

**Назначение:**  
Окно предназначено для генерации случайных паролей с заданными пользователем параметрами.

**Элементы интерфейса:**
- `Label` — текстовые подписи к элементам управления.
- `NumericUpDown` — выбор длины генерируемого пароля.
- `CheckBox` — параметры генерации пароля (использование цифр, строчных и прописных букв, специальных символов).
- `TextBox` — поле для отображения сгенерированного пароля.
- `Button` — кнопки генерации, копирования и очистки результата.

**Логика взаимодействия пользователя:**
1. Пользователь задаёт длину пароля.
2. Выбирает необходимые параметры генерации с помощью чекбоксов.
3. Нажимает кнопку генерации пароля.
4. Сгенерированный пароль отображается в текстовом поле.
5. Пользователь может скопировать пароль или сгенерировать новый.
