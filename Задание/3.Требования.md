# Требования к программному продукту

## 1. Функциональные требования

Программный продукт должен обеспечивать выполнение следующих функций:

- аутентификация пользователя с использованием **мастер-пароля** при запуске приложения;
- добавление новой записи пароля с указанием названия, логина, пароля, категории и заметок;
- редактирование существующих записей паролей;
- удаление выбранных записей паролей;
- генерация случайного пароля с настраиваемыми параметрами:
  - длина пароля;
  - использование цифр;
  - использование специальных символов;
  - использование букв верхнего и нижнего регистра;
- создание новых категорий;
- удаление существующих категорий;
- присвоение записей определённым категориям;
- поиск записей по:
  - названию;
  - категории;
  - логину;
- копирование пароля в буфер обмена с **автоматической очисткой через 30 секунд**;
- экспорт и импорт базы данных в зашифрованном виде;
- автоматическое сохранение всех изменений в зашифрованный файл.

---

## 2. Нефункциональные требования

### 2.1 Требования к производительности

- время загрузки базы данных — **менее 2 секунд**;
- время генерации пароля — **менее 1 секунды**;
- поддержка работы минимум с **1000 записями** без заметного снижения производительности.

### 2.2 Требования к безопасности

- использование алгоритма шифрования **AES-256** для защиты данных;
- хэширование мастер-пароля с применением алгоритма **PBKDF2**;
- защита от перебора пароля (brute-force) путем введения задержки при неверном вводе мастер-пароля;
- хранение данных исключительно в зашифрованном виде.

### 2.3 Требования к интерфейсу

- интуитивно понятный графический интерфейс пользователя (GUI);
- поддержка горячих клавиш для основных операций;
- наличие **светлой и тёмной темы** оформления;
- корректная обработка пользовательских ошибок (например, неверный мастер-пароль).

### 2.4 Требования к надежности и сопровождению

- автоматическое создание резервной копии зашифрованного файла базы данных;
- устойчивость к сбоям и корректная обработка исключительных ситуаций;
- предотвращение потери данных при аварийном завершении работы приложения.

### 2.5 Требования к совместимости и доступности

- поддержка операционных систем **Windows 10 и Windows 11**;
- полная независимость от подключения к сети Интернет;
- поддержка экранных читалок;
- наличие высококонтрастного режима для пользователей с ограничениями по зрению.
